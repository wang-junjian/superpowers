# 执行计划

## 概述

加载计划，批判性地审查，执行所有任务，完成时报告。

**开始时宣布：** "我正在使用 executing-plans 技能来实施这个计划。"

**注意：** 告诉你的人类合作伙伴，Superpowers 在可以访问子智能体的情况下效果会好得多。如果在具有子智能体支持的平台（如 Claude Code 或 Codex）上运行，其工作质量会显著更高。如果有子智能体可用，请使用 superpowers:subagent-driven-development 而不是这个技能。

## 流程

### 第 1 步：加载和审查计划
1. 阅读计划文件
2. 批判性地审查——识别对计划的任何问题或疑虑
3. 如果有疑虑：在开始前向你的人类合作伙伴提出
4. 如果没有疑虑：创建 TodoWrite 并继续

### 第 2 步：执行任务

对于每个任务：
1. 标记为 in_progress
2. 完全按照每个步骤执行（计划有小步骤）
3. 按指定运行验证
4. 标记为已完成

### 第 3 步：完成开发

在所有任务完成并验证后：
- 宣布："我正在使用 finishing-a-development-branch 技能来完成这项工作。"
- **必需的子技能：** 使用 superpowers:finishing-a-development-branch
- 遵循该技能来验证测试，呈现选项，执行选择

## 何时停止并寻求帮助

**在以下情况下立即停止执行：**
- 遇到阻碍（缺少依赖项、测试失败、指令不清楚）
- 计划有关键空白无法开始
- 你不理解指令
- 验证反复失败

**询问澄清而不是猜测。**

## 何时重新审视早期步骤

**在以下情况下返回审查（第 1 步）：**
- 合作伙伴根据你的反馈更新了计划
- 基本方法需要重新思考

**不要强行通过阻碍**——停止并询问。

## 记住
- 首先批判性地审查计划
- 完全按照计划步骤执行
- 不要跳过验证
- 当计划说要引用技能时引用
- 遇到阻碍时停止，不要猜测
- 未经用户明确同意，永远不要在 main/master 分支上开始实施

## 集成

**所需的工作流技能：**
- **superpowers:using-git-worktrees** - 必需：在开始前设置隔离的工作区
- **superpowers:writing-plans** - 创建此技能执行的计划
- **superpowers:finishing-a-development-branch** - 在所有任务后完成开发
